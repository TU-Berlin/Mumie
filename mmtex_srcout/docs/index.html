Brief MmTex Overview
MmTex
bin Folder with the executables
docs Documentation folder
etc Configuration file folder
lib Libraries, documentclasses, core files of MmTex
notes Notes, Todo's and other stuff
samples Sample folder
styles CSS and XSLT folder

Folder / File / Script Brief description
All about samples
samples/xhtml Folder where the converted samples will be stored. The XHTML-files need to be created by the 'create_samples' script. There is also a link to the corresponding samples from the 'XDOC index page' To see a list of the xhtml | tex | xml sample-versions please go to 'sample_index'
samples/tex Folder with the TEX-sources of all samples
samples/xml Folder with the XML-files after the conversion of the sources with mmtex
samples/index.xhtml Index file of the sample folder. Will give you a list of all files, including the TEX-sources, the XML-files and the resulting XHTML
utils/create_samples Script to create the sample files. It will convert all sources in the TEX-folder using mmtex and then convert the resulting XML-files to XHTML using mmxalan. The 'sample_index' is created by this script, too.
All about xdocs
The so called 'xdocs' are files created automatically with mmtex. Every library has its own, build in documentation. One way is to ask mmtex for information on a special command using for example: "mmtex -c addtocounter -l counter" or "mmtex --list-cmds --lib=counter" But the more comfortable way is to browser through the XDOC folder and take a look at the files for every library. All commands are listed, with a short explanation.
docs/xdocs Parent folder of the xdocs, contains the xml_sources folder, where the xml files are located. Also place of the xhtml folder, if XDOC files had been converted to xhtml (may be set as option for the script xdocs_create)
docs/xdocs/xml_sources Folder with the xml_sources of the XDOC files. They may be viewed directly in a browser, because there is a stylesheet associated to every file. But sometimes it's faster to view the xhtml files, especially in case of big xml-files
docs/xdocs/xhtml Folder with the xhtml version of the XDOC files created by xdocs_create. May be a better way to view the content if your browser doesn't support the 'on the fly' conversion or the files are really large, so it would take to long until you'll see anything in your browser.
xdocs/index.xhtml Index file of the xdoc folder. Will present a list of all libraries where XDOCs exist. Provide links to the detailed XHTML-version for every library and the sample file if there is any.
utils/xdoc_create Script to create the XDOCs for all libraries and DCLs. Please call "xdoc_create --help" for more information
All about utils
In this folder all useful scripts are located. Some description below may appear two times, once here and once in the according area. (create_samples e.g is described in the samples section too) The pretty html version of all scripts, you see when clicking on a file, is created by htmlize and create_sub_overview. To know more, please see the source_docs area.
utils/create_samples Script to create the sample files. It will convert all sources in the TEX-folder using mmtex and then convert the resulting XML-files to XHTML using mmxalan. The 'sample_index' is created by this script, too.
utils/xdoc_create Script to create the XDOCs for all libraries and DCLs. Please call "xdoc_create --help" for more information
utils/htmlize Folder where a xemacs '.el' file is located. It may be used to convert all perl files of an directory to html version, so the could be viewed by any browser in a pretty form. It's used by sub_overview to create the html files and the navigation. See there for more details on this.

For stand-alone usage, if there might be a folder with perl sources to convert to html please use:
xemacs -l ~/mmtex/utils/htmlize/htmlize_whole_directory.el -eval '(htmlize-tree("~/mmtex/")' Results will be placed in /docs/source_docs

All about source_docs
In this folder, all HTML-versions of the perl files are located.. Below this folder, the complete folder structure of mmtex is rebuilded, and all folders where normally perl scripts are located, are now filled with the html versions of these scripts.
docs/source_docs/index.xhtml Index file for source_docs, presents a tree to navigate through the HTML-versions of all perl files.
