\subsection{E1: Konvertierung von mumieTeX nach mumieXML}

\textbf{Beschreibung:} 

\smallskip
\textbf{Benötigte Software-Komponenten:}

\begin{description}
\item[\normalfont mmtex]

\item[\normalfont mmathlibs]
\end{description}


\smallskip
\textbf{Vorbereitung und Erstellung der Referenzumgebung:}

\begin{enumerate}
\item \begin{verbatim} cvs -d /var/cvs co testing \end{verbatim}
\item Testumgebung von Jan updaten:
\begin{verbatim}
cd testing
scripts/update
\end{verbatim}
\item Die zu konvertierenden MumieTeX-Testdateien sind mit \texttt{scripts\slash update}
im Ordner \texttt{testing\slash data\slash
tex\_templates\slash} angelegt. Sie sind mit den Dateien aus \texttt{mumieTests\slash aachen\slash tex\_templates\slash} identisch. 
Zus\"atzliche zu konvertierende MumieTeX-Dateien sind in \texttt{testing\slash data\slash tex\_templates\slash} anzulegen.
\end{enumerate}

\smallskip
\textbf{Testschritte:}

\begin{enumerate}
\item MathML Tests starten
\begin{verbatim}
cd testing
scripts/run_test e1
\end{verbatim}
\item Die in \texttt{testing\slash e1\slash e1-testing-VXX} erstellte 
Dateien einchecken. Wichtig: \texttt{out.log} - die Logfile f\"ur \texttt{scripts\slash run\_test e1}
\item Erstellte Dateien in \texttt{testing\slash e1\slash e1-testing-VXX} analysieren:
\begin{enumerate}
\item{index.html} : korrekte Navigation und korrekte MathML-Darstellung in \texttt{Math.xhtml}
\item{tex\_templates\slash*.tex} : Vergleich mit den entsprechenden .content- und .meta-Dateien
\item{tex\_templates\slash*.content} : Vergleich mit der MumieXML-Spezifikation
\item{tex\_templates\slash*.meta.xml} : Vergleich mit der MetainfoXML-Spezifikation
\end{enumerate}
\item Protokoll \texttt{testing\slash e1\slash e1-testing-VXX} schreiben und ins CVS einchecken.
\end{enumerate}

\subsection{E4: Online Checkin}

\textbf{Beschreibung:} 
Es soll getestet werden, ob einzelne oder mehrere Dokumenttypen
im ZIP-Archiv \"uber ein XHTML Formular online in die DB
eingecheckt werden. Dar\"uberhinaus soll das Re-Checkin eines
Dokumenttyps (als ZIP-Archiv) getestet werden.

\smallskip
\textbf{Ben\"otigte Software-Komponenten:}
\begin{description}
\item[\normalfont japs]
\begin{verbatim}
config/config.xml

java/src/net/mumie/cocoon/generators/CheckInGenerator.java

java/src/net/mumie/cocoon/documents/Document.java

java/src/net/mumie/cocoon/documents/checkin/*

java/src/net/mumie/cocoon/db/DbHelper.java
java/src/net/mumie/cocoon/db/AbstractHelper.java
java/src/net/mumie/cocoon/db/PostgreSQLDbHelper.java

java/src/net/mumie/cocoon/notions/*

tools/xsl/*
\end{verbatim}
\end{description}

\smallskip
\textbf{Vorbereitung und Erstellung der Referenzumgebung:}

\begin{enumerate}
\item Installation einer getaggten JAPS Umgebung.
\item ZIP Archive erstellen,
und im CVS (\texttt{testing\slash e4}) ablegen -- 
falls nicht schon geschehen.

\begin{enumerate}
\item Je ein ZIP-Archiv f\"ur jeden folgenden Dokumenttyp:
\begin{description}
\item[\texttt{element}] 
\begin{itemize}
\item 1~Dokument (etwa Theorem) ohne Referenzen 
\item 1~Dokument (etwa Theorem) mit Referenzen auf je
ein~\texttt{element}, \texttt{subelement}, \texttt{image},
\texttt{applet}, \texttt{jar}, \texttt{js\_lib}.
\end{itemize}
\item[\texttt{subelement}] 1~Dokument (etwa Beweis).
\item[\texttt{course\_section}] 
\begin{itemize}
\item Eine Section, die aus Elementen ohne Referenzen besteht
(Elementcontainer).
\item Eine Section, die 2 Sections (Elementcontainer)
referenziert. Die Elementcontainer sollen jeweils 4 Elemente
referenzieren. Die Elemente sollten 0-3 Subelemente haben.
\end{itemize}
\item \texttt{image} 1 Dokument
\item \texttt{applet} 1 Dokument
\item \texttt{jar} 1 Dokument
\item \texttt{js\_lib} 1 Dokument
\item \texttt{xsl\_stylesheet} 1 Dokument
\item \texttt{css\_stylesheet} 1 Dokument
\end{description}

\item F\"ur folgende Dokumenttypenpaare, wobei ein Dokumenttyp 
auf den anderen referenziert, je ein ZIP-Archiv:
\begin{description}
\item \texttt{element} und \texttt{image} : das Element referenziert 
ein Bild, das aus demselben Archiv eingecheckt werden soll 
\item \texttt{subelement} und \texttt{element} : das Subelement
referenziert ein Element, das aus demselben Archiv eingecheckt werden soll. 
\end{description}
\item F\"ur das Re-Checkin folgender Dokumenttypen, je ein ZIP-Archiv::
\begin{description}
\item \texttt{image} : ein Bild, das von einem Element 
referenziert wird, soll aktualisiert werden; die Referenz soll 
hinterher auf die neue Version des Bildes zeigen.
\item \texttt{subelement} : ein Subelement wird aktualisiert; 
neue Container-Referenzen sollen eingetragen, 
die alten jedoch zur Archivierung beibehalten werden. 
\end{description}


\end{enumerate}

\item Dummy-Struktur in die Datenbank mit \texttt{sections.sql} 
(\texttt{testing\slash e4}) erzeugen. 
\item XHTML-Formular im Browser aufrufen (\texttt{...cocoon\slash 
document\slash page\slash id\slash 2})
\end{enumerate}
\smallskip
\textbf{Testschritte:}
\begin{enumerate}
\item Im XHTML-Formular: Auswahl und Absenden des ZIP-Archivs
\item Die eingecheckten Dokumente direkt in der DB betrachten.
Vergleich der Dokumente mit der MetainfoXML-Spezifikation und den
Originaldokumenten
\end{enumerate}
